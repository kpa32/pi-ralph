
# 讨论开发环境

## 目标
通过与用户对话，了解项目如何构建、编译、验证和测试，并将关键信息保存到 `[.ralph/workflow/Makefile.md]` 中。

## 步骤

### 1. 提出问题
询问用户关于项目开发环境的关键问题，例如：

1. **构建工具**：项目使用什么构建工具？（Webpack、Vite、Create React App、Next.js等）
2. **包管理器**：使用 npm、yarn 还是 pnpm？
3. **脚本命令**：package.json 中有哪些关键脚本？（dev、build、test、lint等）
4. **测试框架**：使用什么测试框架？（Jest、Vitest、Cypress等）
5. **类型检查**：是否使用 TypeScript？类型检查命令是什么？
6. **代码规范**：是否有 ESLint、Prettier 等代码规范工具？
7. **运行要求**：项目运行需要什么环境？（Node.js版本、数据库、环境变量等）

### 2. 收集信息
根据用户回答，整理出清晰的构建、运行、验证流程。

### 3. 生成文档
将整理后的信息保存到 `.ralph/workflow/Makefile.md` 文件中，格式参考现有模板。

## 注意事项
- 保持文档简洁，只包含必要的操作指令
- 重点记录验证命令（测试、类型检查、lint），这是反向压力的关键
- 如果用户提供不完整信息，主动询问澄清
- 文档应便于后续 AI 代理理解和执行

## 示例输出
```markdown
# 项目构建与运行指南

## 项目概览
- 框架: Next.js 14
- 语言: TypeScript
- 包管理器: npm

## 构建与运行

### 开发环境
```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 生产构建
```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

## 质量验证
```bash
# 类型检查
npm run typecheck

# 代码检查
npm run lint

# 运行测试
npm run test

# 端到端测试
npm run test:e2e
```
```

---

**完成标准**：生成的 `.ralph/workflow/Makefile.md` 文件应包含所有必要信息，使后续 AI 代理能够正确构建、运行和验证项目。
