

# 项目构建与运行指南

## 项目概览

- 源代码目录: `src/`
- 共享库目录: `src/lib/` (项目标准库，共享工具和组件)
- 配置文件: 根据项目类型而定 (package.json, tsconfig.json, vite.config.ts 等)
- 构建输出: 通常在 `dist/` 或 `build/` 目录

## 构建与运行

### 开发环境
```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 生产构建
```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

## 质量验证（反向压力 Backpressure）

每次代码更改后必须运行以下验证，确保代码质量：

```bash
# 类型检查（TypeScript）
npm run typecheck

# 代码风格检查
echo "根据项目实际配置执行"

# 运行测试
npm test

# 如果存在端到端测试
npm run test:e2e
```

**重要**：所有验证必须通过才能提交代码。如果测试失败，必须修复问题后再继续。

## 测试策略

- **单元测试**：测试独立函数和组件
- **集成测试**：测试模块间交互
- **端到端测试**：测试完整用户流程
- **快照测试**：确保UI不变

## Ralph 工作流程集成

### 计划模式（Planning Mode）
- **输入**：`.ralph/specs/*` 规格说明书
- **输出**：`BUILD_PLAN.md` 开发计划
- **提示词**：`ralph_plan.md` + 本文件
- **执行方式**：启动单次迭代循环，分析规格与代码差距，生成计划

### 构建模式（Build Mode）
- **输入**：`BUILD_PLAN.md` 开发计划
- **输出**：实现的功能代码
- **提示词**：`ralph_build.md` + 本文件
- **执行方式**：启动循环迭代，每次完成一个任务，直到计划全部完成

### 循环控制
- 每次迭代处理一个任务，保持上下文清洁
- 任务完成后自动提交代码
- 依赖磁盘文件传递状态（specs、BUILD_PLAN.md、本文件）
- 遵循"计划可丢弃"原则：计划错误时可随时重新生成

### 质量保证
- **反向压力**：测试、类型检查、lint 必须通过
- **完成标准**：所有验证通过，功能完全实现
